# Investigation Process

## Step 1 – Review the Alert

Event ID 4720 indicated that a new local user account had been created.

The investigation focused on identifying the account that was created and determining which user performed the action.

---

## Step 2 – Identify the New Account

The **New Account** section identified the newly created account as:

**soclab**

The account name matched the test account created during the lab exercise.

**Finding:** Expected account creation.

---

## Step 3 – Identify the Actor

The **Subject** section identified the account responsible for creating the new user.

**Account Name:** kevin

This confirmed that the logged-in administrator created the account.

**Finding:** Authorized administrative action.

---

## Step 4 – Verify Event Details

The event timestamp matched the time the test account was created.

No unexpected user accounts or unauthorized administrative activity were identified.

**Finding:** Activity consistent with expected laboratory testing.

---

# Analysis

The investigation confirmed that the account creation event was generated by the logged-in administrator during a controlled lab exercise.

Although Event ID 4720 can indicate persistence techniques used by attackers, no evidence suggested unauthorized activity in this case.

---

# Conclusion

The newly created account was determined to be legitimate.

The account was intentionally created for training purposes and was successfully attributed to the authorized user.

**Final Disposition:** Closed

---

# Analyst Notes

This investigation demonstrated how Windows Security Event ID 4720 records local account creation events. The event provides valuable forensic information, including the account that performed the action, the newly created account, and the exact time of creation.

This lab also highlighted the importance of enabling **User Account Management** auditing before attempting to investigate account creation events.
